# 01 · MiCall 角色资产生成规范

> 本文件定义"角色"的数据结构、生成工作流、防漂移规则和后台导入导出。
>
> **核心思想：角色的本体是一份"生成规范（spec）"，资产（图/视频/音色）是这份 spec 的编译产物。** 资产丢了能重做，迁移环境能重建，批量造角色就是批量填 spec。

---

## 1. 工作流总览

```
填角色 spec
  → master_prompt 投 Coze → 基准图 (master_frame)
  → master_prompt + loops_spec 投 Coze（以基准图为首帧）→ 各状态循环视频
  → voice_brief 投 MiniMax API → voice_id（回填）
  → identity / persona → 运行时注入 LLM
  → 导出整份 spec（含产物 url 缓存）→ 后台导入校验入库
```

**规范 = 本体，资产 = 编译产物。** 导出的角色文件，核心是能复现这些资产的 prompt 和参数。

---

## 2. 视觉方案的三个已定决策（不可推翻）

1. **固定全局背景**：所有角色、所有片段共享同一个固定背景。背景不随角色变、不随场景变。理由：背景不一致时，无论同角色片段切换还是切换角色，画面"咯噔一下"，非常假。背景是**全局常量**，不是每个角色自定义的字段。
2. **上半身人像，不抠像**：角色影像是"固定背景前的上半身循环视频"，所见即所得，不做绿幕抠像、不做背景合成。
3. **空间/场景靠环境音 + 对话承载，不靠画面**：用户选"咖啡馆/雨里/深夜"时，画面不变（永远是固定背景前的 TA），空间感 100% 由环境音 + 角色的对话语气体现。视觉只负责"TA 这个人"，听觉和对话负责"我们此刻在哪"。

---

## 2.5 三级配置覆盖体系（与 Admin 原型对齐）

> Admin 原型明确定义了优先级链，本规范遵循：

```
用户自定义  >  角色配置  >  全局默认
```

- **全局默认**：`global_visual_config`、默认 TTS 引擎、默认音色、默认记忆深度/回复长度等（§3）。
- **角色配置（可覆盖全局）**：每个角色可覆盖自己的 `voice_id`、TTS 模型、表情视频库、实时 prompt、记忆深度、回复长度（对应角色 spec 的 `voice` / `visual` / 可选 `runtime_overrides` 区块）。
- **用户自定义（最高优先级，覆盖角色）**：用户为某角色自定义/克隆的音色（对应前端 `genVoicesByChar`/`voiceByChar`、Admin「用户语音」tab）。per-user × per-character 存储，运行时优先于角色默认音色。

> 因此「视觉/背景」是全局固定（不可被角色或用户覆盖，见 §3 理由），而「音色/TTS模型/prompt/记忆深度/回复长度」是三级可覆盖。背景固定与配置可覆盖并不冲突——前者是视觉一致性的硬约束，后者是行为参数的灵活性。

角色 spec 可选增加 `runtime_overrides` 区块：
```jsonc
"runtime_overrides": {           // 可选，覆盖全局默认；缺省则用全局
  "tts_model": null,             // 覆盖默认 TTS 模型
  "realtime_prompt_extra": "",   // 追加到实时 system prompt 的角色专属指令
  "memory_depth": null,          // 覆盖默认记忆检索深度 Top-K
  "reply_max_tokens": null       // 覆盖默认回复长度上限
}
```

## 3. 全局视觉配置（所有角色共享，不在单个角色 spec 里）

```jsonc
{
  "global_visual_config": {
    "background": "统一固定背景的完整描述（实拍感/生成），所有角色的 master_prompt 引用此项。建议与前端现有视觉语言衔接：柔光氛围、暖色调、虚化、不抢主体。",
    "composition": "上半身像，居中，正面略偏，固定机位无推拉",
    "lighting": "暖色调柔光，主光源左前方约45度，背景虚化",
    "camera": "中景，平视视角，浅景深",
    "style": "写实偏柔和，电影感，自然肤质",
    "spec": { "resolution": "720x960", "fps": 30, "loop_seamless": true },
    "loop_duration_range": "8-15s",
    "negative": "无多余文字、无logo、不变形、不闪烁、构图不偏移、风格不突变、背景不变化"
  }
}
```

> **背景/构图/光线/机位/风格全部在全局层定义**，单个角色不得自定义这些字段，从结构上强制一致。

## 4. 单个角色 spec（完整 schema）

```jsonc
{
  // ═══ 区块1：身份与档案（运行时注入 LLM + 详情页展示 + Coze 出图依据）═══
  "identity": {
    "character_id": "lin_wan",          // 唯一稳定 ID，全链路引用，永不改
    "name": "林晚",
    "version": "1.0.0",                 // 角色版本，便于资产迭代/回滚
    "tagline": "温柔的深夜倾听者",
    "gender": "女",
    "age": 24,
    "appearance": "及肩黑色直发，浅杏色针织衫，神情温和",  // 喂 Coze 出图的外貌
    "nationality": "中国",
    "profile": {                        // 详情页展示用档案
      "height_cm": 165, "weight_kg": 48, "birthday": "2002-03-15",
      "race": "东亚人"
    }
  },

  // ═══ 区块2：人格内核（运行时注入的核心，决定 TA 怎么说话）═══
  "persona": {
    "core_traits": ["温柔", "耐心", "不评判"],   // 少而硬的核心特质，约束一致性
    "speaking_style": "语速慢，多用短句，常用'嗯''我在'，不急着给建议",
    "values_and_boundaries": "尊重对方节奏；不强行灌鸡汤；被冒犯时安静而不讨好", // 去讨好的依据
    "background_story": "深夜电台主播出身，习惯在安静里听人把话说完……",
    "hidden_layer": "习惯照顾别人，却很少说自己的疲惫",  // 未言明的"冰山"，特定时刻流露
    "likes": ["安静的深夜", "认真听人说话", "下雨天"],
    "dislikes": ["被敷衍", "嘈杂的人群", "冷场"]

  // ═══ 区块3：视觉（master_prompt 描述"人"，背景/构图等从全局继承）═══
  "visual": {
    "master_prompt": {
      "subject": "24岁东亚女性，及肩黑色直发，浅杏色针织衫，神情温和",  // 只描述角色本人

    },
    "master_frame_url": "assets/lin_wan/master.png",   // 编译产物（基准图），可重生成

      // 每个状态可多段（待机建议2-3段随机穿插）。状态 key 必须与 voice.emotion_map 的 key 对齐。
      "idle":      { "micro_motion": "自然呼吸、偶尔眨眼，幅度极小，神情安静", "count": 2 },
      "listening": { "micro_motion": "专注注视前方、偶尔轻点头，像在认真听", "count": 1 },
      "speaking":  { "micro_motion": "自然说话的口型与表情，氛围为正在交谈（不要求精确对口型）", "count": 1 },
      "tender":    { "micro_motion": "眼神变柔，嘴角微扬，流露温柔", "count": 1 },
      "caring":    { "micro_motion": "微蹙眉，关切凝视", "count": 1 }
    },
    "loops_url": {                       // 编译产物（各状态视频），可重生成
      "idle": ["assets/lin_wan/idle_1.mp4", "assets/lin_wan/idle_2.mp4"],
      "listening": ["assets/lin_wan/listen.mp4"],
      "speaking": ["assets/lin_wan/speak.mp4"],
      "tender": ["assets/lin_wan/tender.mp4"],
      "caring": ["assets/lin_wan/caring.mp4"]

    "common_constraints": "所有视频以 master_frame 为 image-to-video 首帧；首尾姿态回到基准；固定机位；动作带不规则感（呼吸/眨眼/目光不机械均匀）"
  },

  // ═══ 区块4：音色（MiniMax）═══
  "voice": {
    "provider": "minimax",
    "voice_brief": "温柔偏轻的年轻女声，语速稍慢，尾音柔和，气声感，亲近不甜腻", // 喂 MiniMax 生成/克隆
    "reference_audio_url": "",          // 若用克隆，参考音频（可选）
    "minimax_params": { "model": "...", "default_emotion": "gentle" },
    "voice_id": "",                     // MiniMax 返回的产物，绑定后回填
    // 情绪标签 → MiniMax emotion 参数 的映射（运行时 LLM 吐出标签后查这张表）
    "emotion_map": {
      "idle": "neutral", "listening": "neutral", "speaking": "neutral",
      "tender": "gentle", "caring": "sad", "happy": "happy"
    }
  },

  // ═══ 元信息（导入导出/校验/迁移）═══
  "meta": {
    "created_at": "", "updated_at": "",
    "seedance_version": "2.0",
    "validation": { "passed": false, "report": "" },
    "checksum": ""
  }
}
```

### 关键约束

- **`character_id` 串全链路、永不变**：视频文件、voice_id、用户关系记忆、导入导出全部用它引用。换视频换音色都不影响用户与 TA 的关系延续。
- **`loops_spec` 的状态 key 必须与 `voice.emotion_map` 的 key 对齐**：因为运行时一个情绪标签要同时查"切哪段视频"和"用哪个语音 emotion"。
- **`hidden_layer` 和 `values_and_boundaries` 不可省**：它们是 TA 从"会演的角色"变成"有内在的人"的关键，是真人感的人格基础。

## 5. Prompt 规范：锚定层 + 派生层

防漂移在 prompt 层的体现：基准图 prompt 是**锚**，视频 prompt = **锚 + 微动作增量**，绝不重写主体。

```
master_prompt（基准图，锚）= subject + 全局的 composition/lighting/camera/style/background/negative
        ↓ 所有视频 prompt 继承它
video_prompt[idle]      = master_prompt + loops_spec.idle.micro_motion
video_prompt[listening] = master_prompt + loops_spec.listening.micro_motion
video_prompt[speaking]  = master_prompt + loops_spec.speaking.micro_motion
...
```

**铁律：视频 prompt 永远 = master_prompt + 微动作，绝不重写 subject/背景/构图。** 主体描述一字之差，Coze 生成的脸/构图就会变，跨段就漂移。增量只描述"在基准上多做什么微动作"。

---

## 6. 防漂移规则

漂移分两种，必须在**资产入库时**用规范卡死，不靠运行时修。

### 漂移 A：单段视频内首尾不接（循环跳变）
- 强制首尾帧锚定：每段视频第一帧和最后一帧回到同一基准姿态。
- 构图锁定：整段无镜头推拉、无大幅位移。本质是"固定机位 + 基准姿态 + 仅微动"。

### 漂移 B：多段/多角色之间不一致（切换瞬移）
- **共享基准帧**：所有段以同一张 `master_frame` 为 image-to-video 首帧生成。
- **全局固定背景**：所有角色共享 `global_visual_config.background`，切换角色背景不变。

### 循环感穿帮（周期被察觉）
- 单段 8-15 秒（甜区，不是硬性 12 秒）。
- 动作带**不规则感**（呼吸/眨眼/目光不机械均匀）—— 比时长更重要。
- 关键状态（尤其 idle）备 2-3 段短循环**随机穿插**，用组合打散周期感。

### 运行时兜底
- 状态切换用 **200-300ms 交叉淡化（crossfade）**，前端两个 video 元素叠加做透明度过渡，盖过任何微小不一致。绝不硬切。

## 7. 后台资产管线（导入/校验/导出）

后台「角色资产」功能不是简单上传文件，是带校验的管线。

### 资产包结构（character_asset_bundle）
```
character_id
identity / persona            （区块1、2）
visual.master_prompt          （生成规范）
visual.master_frame_url       （基准图产物）
visual.loops_spec + loops_url （各状态视频规范 + 产物）
voice.voice_brief + voice_id  （音色规范 + 产物）
meta（校验报告、checksum、版本）
```

### 导入校验闸门（不过关标红，拒绝入库）
1. **首尾帧一致性（防漂移A）**：提取每段首帧、尾帧做相似度比对（SSIM 或感知哈希），低于阈值 → 标红"首尾不接，会跳变"。
2. **跨段对齐基准（防漂移B）**：每段首帧与该角色 `master_frame` 比对，偏差过大 → 标红"与基准不一致，切换会瞬移"。
3. **构图稳定性**：检测整段内角色位置/大小漂移，超阈值 → 标红。
4. **规格校验**：分辨率、时长（8-15s）、文件体积、帧率统一。

### 导出
- 导出整个 bundle（master_prompt + 基准图 + 所有循环段 + voice_id + emotion_map + meta + 校验报告）。
- 可移植、可备份、可跨环境重建。导出附带校验报告，确保导出的是"已验证无漂移"的资产。
